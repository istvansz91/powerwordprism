Sentiment Analysis Procedure

  • Programming implications and progress

    The project is written with Python programming languages and it uses Vader nltk as its main sentiment analysis tool.

    The first stage was to gather corpora from Blizzard forums using "beautiful soup" python library. Afterwards, the data was structured as JSON files and it was annotated using vader.
    To increase the accuracy and omit negative idea effect, we grouped up posts by authors and got an average for every author.

    In addition to that, more adjustment was done on the data to increase sentiment analysis result accuracy. For example, sentimets between -0.5 to 0.5 -with -1 being lowest and 1 being highest- has been removed in the last scoring, since they were not presenting any significant sentiment in this domain.

    In order to dig deeper into class sentiment analysis to have a more clear idea of each aspect of gameplay, a classification was applied on the posts to categorize them based on PvP and PvE aspects. Later we improved ontology and we were able to do this classification based on instances types in each type of gameplay. This can ultimatly provide game developers with exact point to focus on class designs.



Gold Standard

The accuracy was manually monitored in small stages during classification process, however, after finalizing last stage of annotation, more than a 600 posts were randomly selected from different aspects and were manually annotated. The results were compared with Vader Sentiment Analysis annotations to provide the last classification accuracy.
